1. VPCForDAP — all networking (subnets, endpoints, security groups)

3. GlobalLogBucket — access logging for all S3 buckets
4. LambdaSecurityGroup — all lambdas and VPC endpoints
5. SubnetForDAP1/2/3 — all lambdas, endpoints, and Redshift
6. ELTMetadataBucket — state machines, Glue jobs, lambdas
7. DAPNotificationsTopic — all alarms and error notifications
8. RedshiftSecret — Redshift workgroup, rotation, flyway, state machines
9. GlueSecurityConfig — all Glue jobs and crawlers
10. GlueScriptsExecutionRole — all Glue jobs

Circular/Mutual Dependencies

There's one notable circular reference pattern:

- DAPAnalystsFilesBucketPolicy → CloudFrontDistribution → DAPAnalystsFilesBucket → ... and DAPAnalystsFilesBucketPolicy → DAPAnalystsFilesBucket

CloudFormation handles this because the bucket policy is a separate resource that references both the bucket and distribution, but doesn't block either's creation.

There's also a state machine chain:
TxmaRawLayerConsolidatedSchemaProcessingStateMachine → RedshiftConsolidatedModelProcessingStateMachine → ADMConsolidatedModelProcessingStateMachine → ADMV3RefreshStateMachine

This forms a linear invocation chain, not a cycle.
